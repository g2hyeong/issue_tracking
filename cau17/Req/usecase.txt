유스케이스 명
	관리자 로그인
액터 명
	주 액터: Admin
	부 액터: System
개요
	시스템에 진입하기 위해 로그인하여 관리 기능을 수행하는 서비스이다.
사전조건
	- 관리자 계정이 존재해야 한다.
	- 관리자가 로그인되지 않은 상태여야 한다.
사후조건
	- 관리자가 로그아웃 할 때까지 로그인한 상태를 유지한다.
기본 흐름
	1. 관리자는 로그인 페이지에 접근하여 아이디와 비밀번호를 입력한다.
	2. 시스템은 입력된 정보를 검증하고 성공하면 관리자 페이지로 이동한다.
대체 흐름
	- 관리자 아이디 또는 비밀번호를 잘못 입력하면 오류 메시지가 표시되고 다시 입력하도록 한다.
	- 사용자가 입력한 정보에 빠진 필수 정보가 있다면 메시지를 띄워 전부 입력할 수 있도록 한다.
	



유스케이스 명
	프로젝트 생성
액터 명
	Admin
개요
	관리자는 새로운 프로젝트를 생성할 수 있다. 프로젝트는 프로젝트명에 의해 유일하게 식별되므로, 생성 시 프로젝트 명의 중복을 허용하지 않는다.
사전조건
	- 관리자는 관리 권한을 가진 ID로 로그인하고, 프로젝트 생성 기능을 실행한다.
사후조건
	- 새로운 프로젝트가 생성되고 프로젝트 목록에 추가된다.
기본 흐름
	1. 관리자는 프로젝트 생성 페이지에 접근한다.
	2. 관리자는 프로젝트 이름을 입력한다.
	3. 시스템은 입력된 정보를 검증한다.
	4. 검증이 성공하면 새로운 프로젝트가 생성되고 프로젝트 목록에 추가된다.
대체 흐름
	- 필수 입력 항목을 입력하지 않으면 오류 메시지가 표시되고 다시 입력하도록 한다.
	



유스케이스 명
	계정 관리
액터 명
	주 액터: Admin
	부 액터: System
개요
	관리자는 시스템의 계정을 생성, 초기화, 삭제하고, 시스템은 관리자의 작업을 지원한다.
사전조건
	- 관리자가 로그인이 되어 있어야 한다.
사후조건
	- 계정 생성/초기화/삭제 작업이 성공적으로 완료되었을 경우, 해당 작업에 대한 결과가 시스템에 반영된다.
기본 흐름
	1. 관리자가 시스템 내에서 계정 관리를 시작한다.
	2. 관리자가 계정 생성/초기화/삭제 중 하나의 작업을 선택한다.
	3. 선택한 작업에 따라 필요한 정보를 입력하고 확인한다.
	4. 시스템은 관리자가 요청한 작업을 처리하고, 결과를 반영한다.
대체 흐름
	- 필수 입력 항목을 입력하지 않으면 오류 메시지가 표시된다.
	- 이미 존재하는 사용자 이름을 사용하면 오류 메시지가 표시된다. (계정 생성 시)
	- 삭제하려는 계정에 소속된 이슈가 있으면 오류 메시지가 표시되고 계정 삭제를 진행할 수 없다. (계정 삭제 시)
	- 초기화할 사용자가 존재하지 않는 경우 오류 메시지가 표시되고 계정 초기화를 진행할 수 없다. (계정 초기화 시)
	



유스케이스 명
	계정 생성
액터 명
	주 액터: Admin
	부 액터: System, User
개요
	관리자가 시스템에 새로운 사용자 계정을 추가하는 작업이다.
사전조건
	- 관리자가 로그인된 상태여야 한다.
	- 사용자는 db에서 중복되지 않는 아이디를 만들 수 있다.
사후조건
	- 새로운 사용자 계정이 시스템에 추가되었을 경우, 해당 계정으로 로그인할 수 있어야 한다.
기본 흐름
	1. 관리자가 시스템 내에서 계정 생성을 시작한다.
	2. 필수 정보(사용자 이름, 비밀번호 등)를 입력한다.
	3. 시스템은 새로운 사용자 계정을 생성하고, 성공적으로 생성되었음을 알린다.
	4. 관리자는 새로운 사용자 계정으로 로그인할 수 있게 된다.
대체 흐름
	- 관리자가 필수 정보(사용자 아이디, 비밀번호)를 모두 입력하지 않고 계정 생성을 시도할 경우, 시스템은 필수 정보를 입력하도록 안내하고, 계정 생성 작업을 다시 시작하도록 한다.
	- 이미 존재하는 사용자 이름을 사용하면 오류 메시지가 표시되고 다시 입력하도록 한다.
	



유스케이스 명
	계정 초기화
액터 명
	주 액터: Admin
	부 액터: System
개요
	관리자가 시스템 내에서 사용자 계정을 초기화하는 작업이다.
사전조건
	- 관리자가 로그인된 상태여야 한다.
	- 초기화하려는 계정이 존재해야 한다.
사후조건
	- 선택한 사용자 계정의 정보가 초기화되었을 경우, 해당 사용자는 시스템에 다시 로그인해야 한다.
기본 흐름
	1. 관리자가 시스템 내에서 계정 초기화를 시작한다.
	2. 초기화할 사용자 계정이 존재하는지 확인한다.
	3. 시스템은 선택한 사용자 계정의 정보를 초기화하고, 성공적으로 초기화 되었음을 알린다.
대체 흐름
	- 관리가자 초기화할 사용자 계정을 선택하지 않고 초기화를 시도했을 경우, 시스템은 사용자 계정을 선택하도록 안내하고, 계정 초기화 작업을 다시 시작하도록 한다.
	- 시스템 오류로 인해 계정 초기화 작업이 실패했을 경우, 시스템은 오류 메시지를 표시하고, 계정 초기화 작업을 다시 시작하도록 한다.
	



유스케이스 명
	계정 삭제
액터 명
	주 액터: Admin
	부 액터: System
개요
	관리자가 시스템 내에서 사용자 계정을 삭제하는 작업이다.
사전조건
	- 관리자가 로그인된 상태여야 한다.
	- 삭제하려는 계정이 존재해야 한다.
	- 삭제하려는 계정에 소속된 이슈가 없어야 한다.
사후조건
	- 사용자 계정이 삭제된다.
	- 선택한 사용자 계정이 삭제되었을 경우, 해당 사용자는 시스템에 더 이상 접근할 수 없어야 한다.
기본 흐름
	1. 관리자가 시스템 내에서 계정 삭제를 시작한다.
	2. 시스템은 선택된 계정이 존재하는지 확인한다.
	3. 삭제할 사용자 계정을 선택하고 확인한다.
	4. 시스템은 선택한 사용자 계정을 삭제하고, 성공적으로 삭제되었음을 알린다.
	5. 삭제를 취소하면 이전 단계로 돌아간다.
대체 흐름
	- 선택된 계정이 존재하지 않으면 오류 메시지가 표시된다.
	- 삭제하려는 계정에 소속된 이슈가 있으면 오류 메시지가 표시되고 계정 삭제를 진행할 수 없다.
	



유스케이스 명
	이슈 검색
액터 명
	User (Tester, PL, Dev)
개요
	이슈 검색은 사용자가 시스템 내의 이슈를 검색하는 작업이다.
사전조건
	- 사용자가 로그인되어 있어야 한다.
사후조건
	- 검색 결과에 따라 사용자는 해당 이슈들을 확인할 수 있어야 한다.
기본 흐름
	1. 사용자는 이슈 검색 페이지에 접근한다.
	2. 사용자는 검색 조건을 입력한다.
	3. 시스템은 입력된 검색 조건에 해당하는 이슈를 검색한다.
	4. 검색 결과가 있으면 이슈 목록을 표시한다.
	5. 검색 결과가 없는 경우 적절한 메시지가 표시된다.
대체 흐름
	- 입력된 조건에 해당하는 이슈가 없을 경우, 시스템은 메시지를 표시한다.
	- 사용자가 검색 조건을 입력하지 않고 검색을 시도할 경우, 시스템은 검색 조건을 입력하도록 안내한다.

유스케이스 명
	이슈 상태 변경
액터 명
	User (Tester, PL, Dev)
개요
	사용자가 이슈의 상태를 변경한 내용이 시스템에 반영되어야 한다.
사전조건
	- 사용자가 로그인되어 있어야 한다.
	- 상태를 변경할 이슈가 존재해야 한다.
	- 변경하려는 상태가 유효해야 한다.
사후조건
	- 사용자가 이슈의 상태를 변경한 내용이 시스템에 반영되어야 한다.
기본 흐름
	1. 사용자가 상태를 변경할 대상 이슈를 선택한다.
	2. 사용자는 이슈의 상태를 변경할 새로운 상태로 선택한다.
	3. 시스템은 사용자가 선택한 상태로 이슈의 상태를 변경한다.
대체 흐름
	- 사용자가 존재하지 않는 이슈의 상태를 변경하려고 시도할 경우, 시스템은 사용자에게 존재하지 않는 이슈임을 알리고, 상태 변경 작업을 중지한다.
	- 사용자가 이슈의 상태를 변경할 수 있는 옵션이 충분하지 않은 경우, 시스템은 사용자에게 상태 변경 옵션을 더 제공하도록 안내한다.

유스케이스 명
	사용자 로그인
액터 명
	주 액터: User (Tester, PL, Dev)
부 액터: Admin
개요
	시스템에 접근하기 위해 사용자가 자신의 계정으로 로그인하여 이슈 관리 기능을 사용한다.
사전조건
	- 사용자가 시스템에 등록된 유효한 계정 정보를 가지고 있어야 한다.
	- 사용자가 로그인되지 않은 상태여야 한다.
사후조건
	- 사용자가 성공적으로 로그인한 후에는 시스템의 기능을 사용할 수 있어야 한다.
기본 흐름
	1. 사용자는 로그인 화면에 접근한다.
	2. 사용자는 자신의 계정 정보(아이디, 비밀번호)를 입력한다.
	3. 시스템은 입력된 계정 정보가 올바른지 확인하고, 사용자의 권한에 맞는 화면을 제공한다.
	4. 사용자는 로그인에 성공하면 시스템의 기능을 사용할 수 있다.
대체 흐름
	- 사용자가 올바르지 않은 계정 정보로 로그인을 시도 할 경우, 시스템은 로그인 실패 메시지를 표시하고, 사용자는 다시 로그인을 시도해야 한다.
	- 사용자가 입력한 정보에 빠진 필수 정보가 있다면 메시지를 띄워 전부 입력할 수 있도록 한다.
	



유스케이스 명
	이슈 생성
액터 명
	Tester
개요
	테스터가 시스템에 새로운 이슈를 등록한다.
사전조건
	- 테스터가 시스템에 로그인된 상태여야 한다.
	- 이슈 생성 권한이 있어야 한다.
사후조건
	- 테스터가 성공적으로 이슈를 생성한 후, 해당 이슈가 시스템에 등록되어야 한다.
기본 흐름
	1. 테스터는 이슈 생성 화면으로 이동한다.
	2. 테스터는 새로운 이슈의 정보(제목, 설명 등)를 입력한다.
	3. 테스터는 이슈를 제출하고 등록한다.
	4. 시스템은 입력된 정보를 확인하고, 새로운 이슈를 시스템에 등록한다.
대체 흐름
	- 테스터가 필수 정보를 입력하지 않고 이슈를 제출하려고 시도할 경우, 시스템은 필수 정보를 입력하도록 안내하고, 이슈 생성 작업을 중지한다.
	



유스케이스 명
	이슈 통계 분석
액터 명
	System
개요
	시스템은 발생한 이슈에 대한 통계 정보를 분석하여 관리자에게 제공한다.
사전조건

기본 흐름
	1. 시스템은 이슈 데이터를 수집한다.
	2. 시스템은 분석 대상 기간 및 조건에 따라 수집된 데이터를 분석하여 통계 정보를 생성한다.
	3. 생성된 통계 정보를 시스템이나 사용자에게 제공한다.
대체 흐름
	- 시스템이 이슈 데이터를 수집하거나 분석하는 과정에서 오류가 발생하면 시스템은 오류 메시지를 표시하고, 통계 분석 작업을 중지한다.
	- 분석 대상 기간 또는 조건이 정의되지 않은 경우, 오류 메시지가 표시되고 통계 분석 작업을 중지한다.

	배정자 자동 추천
액터 명

개요
	시스템은 새로운 이슈가 등록될 때, 이전 이슈의 이력을 분석하여 적합한 배정자를 추천한다.
사전조건
	- 새로운 이슈가 생성되어야 한다.
	- 이전에 해결된 유사 이슈가 존재하고 시스템은 이전 이슈 데이터를 적절히 수집하고 저장해야 한다.
사후조건
	- 시스템은 새로운 이슈에 대한 배정자를 자동으로 추천할 수 있어야 한다.
기본 흐름
	1. 새로운 이슈가 등록되면 시스템은 이전 이슈 데이터를 분석한다.
	2. 분석 결과를 바탕으로 시스템은 적절한 배정자를 추천한다.
	3. 시스템은 추천된 배정자를 사용자에게 표시한다.
대체 흐름
	- 시스템이 이전 이슈 데이터를 분석하는 과정에서 유사 이슈를 찾지 못하는 경우, 시스템은 이슈의 특성에 따라 적절한 추천이 불가능하므로 배정자를 추천하지 않고 사용자에게 표시한다.
